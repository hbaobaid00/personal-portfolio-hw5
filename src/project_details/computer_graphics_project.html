Software Rasterizer & Blender Rendering Project

This project involved extending a provided CPU-based rendering framework to implement key components of a rasterization pipeline. Across multiple homework assignments in CSE 167 (Computer Graphics), I configured rendering behavior, implemented geometry shading logic, and produced shaded images using models authored and exported from Blender.

Rendering Features Worked On

  • Triangle rasterization and per-pixel evaluation
  • Barycentric interpolation for pixel depth and surface shading
  • CPU-side lighting evaluation using the provided Phong model utilities
  • Directional, point, and spotlight illumination setup
  • Scene configuration using starter matrix routines
  • Importing PLY files exported from Blender into the homework renderer

The framework supplied core mathematical utilities and loaders; my work focused on integrating scene definitions, debugging shading behavior, adjusting light/material settings, and producing rendered output images entirely on CPU ‚Äî without GPU acceleration or OpenGL.

Workflow Overview

The first two objects were authored in Blender (the donut and table model), exported as PLY files, and rendered using the homework rasterizer. The PLY files for the other objects were given to me by my professor. I tuned light colors, shininess parameters, transformation values, and viewing configurations to develop visually distinct results.

Rendered Outputs

These outputs show successful control over geometry ingestion, illumination response, and per-pixel shading ‚Äî demonstrating foundational understanding of graphics pipelines used in modern engines.
